# 🏗️ 2. FINAL SYSTEM DESIGN (How everything connects)

```
User Browser
     ↓
Azure Static Web Apps (Frontend)
     ↓ (REST API calls)
Azure Functions (Backend API)
     ↓
Cosmos DB (metadata + users)
     ↓
Blob Storage (images)
```

Optional:

* Azure Front Door → improves performance + routing

---

# 🎯 3. FEATURE IMPLEMENTATION (Mapped to Azure)

## 👤 Creator Flow

1. Upload image → stored in Blob Storage
2. Metadata → saved in Cosmos DB
3. API handles validation

## 👥 Consumer Flow

1. Fetch feed → Cosmos DB
2. Load images → Blob Storage (via CDN)
3. Comment + rate → stored in DB

# 🔌 5. BACKEND (Azure Functions)

Use **HTTP-triggered functions**.

### API Endpoints

| Function                  | Method | Purpose       |
| ------------------------- | ------ | ------------- |
| `/api/images`             | GET    | Get feed      |
| `/api/images/{id}`        | GET    | Image details |
| `/api/images`             | POST   | Upload image  |
| `/api/search`             | GET    | Search images |
| `/api/comments`           | POST   | Add comment   |
| `/api/comments/{imageId}` | GET    | Get comments  |
| `/api/rate`               | POST   | Rate image    |

# 🖼️ 6. IMAGE HANDLING (Blob Storage)

### Setup

* Create **container**: `images`
* Set access: public (or SAS tokens)

### Upload Flow

1. Frontend → API
2. API → Blob Storage
3. Save URL in Cosmos DB

### Optimization (important for marks)

* Store:

  * Original image
  * Thumbnail version
* Use:

  * Azure Functions for resizing
